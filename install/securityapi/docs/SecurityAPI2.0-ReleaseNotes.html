Jakarta EE Security Technology Compatibility Kit, Version 2.0

Release Notes, May 2021

Kit Contents

The Jakarta EE Security, Technology Compatibility Kit (TCK) Version 2.0, includes the following items:
  • Jakarta EE Security TCK documentation:
    • These Release Notes can be found with this TCK bundle. If changes or additions are required, you will find them at the Jakarta EE Security Specification web site: https://jakarta-ee/specifications/security/2.0/
  • Jakarta EE Security TCK Signature, API, and specification tests:
    • A signature test that checks that all of the public APIs are supported or defined as specified in the Jakarta EE Security Version 2.0 implementation under test.
    • API tests for the public APIs and language elements defined by the Jakarta EE Security 2.0 Specification.
    • Specification tests for the testable assertions required by the Jakarta EE Security 2.0 Specification.

Platform Notes

The Jakarta EE Security TCK tests have been built with JDK 8 (1.8) and tested with a Jakarta EE Compatibile Implementation Eclipse GlassFish 6 or 6.1 running on the JavaTM Platform, Standard Edition 8 or 11. If you are using a different Web server, you will need to make adjustments to the procedures outlined in this document.

The Jakarta EE Security tests run on the following platforms:

  • CentOS Linux 7
  • Alpine Linux v3.12

Installing, Setting Up, and Running the Jakarta EE Security TCK

Refer to the Jakarta EE Security Technology Compatibility Kit 2.0 User's Guide (HTML, PDF) for complete instructions on installing, setting up, and running the Jakarta EE Security TCK.
